HUB-10 E2E Verification Sprint

Mục tiêu:
Không kiểm tra route tồn tại hay 401 auth nữa.

Phải chứng minh toàn bộ Social Graph hoạt động end-to-end với dữ liệu thật.

==================================================
REQUIREMENT
===========

Tạo 2 user test:

social-user-a
social-user-b

Có JWT riêng cho mỗi user.

Tất cả proof phải chạy bằng dữ liệu thật trong DB.

Không dùng mock.

==================================================
PROOF 1 — FRIEND REQUEST
========================

User A gửi lời mời kết bạn cho User B.

POST /api/social/friends/request

Expected:

201 Created

Response chứa:

requestId
fromUserId
toUserId
status=PENDING

Sau đó:

GET /api/social/friends/pending

User B phải nhìn thấy request.

==================================================
PROOF 2 — ACCEPT FRIEND REQUEST
===============================

User B accept request.

POST /api/social/friends/:requestId/accept

Expected:

200 OK

Sau đó:

GET /api/social/friends

User A thấy User B.

User B thấy User A.

Kiểm tra DB:

social_relationships

phải tạo 2 bản ghi FRIEND.

==================================================
PROOF 3 — DECLINE FRIEND REQUEST
================================

Tạo request mới.

User B decline.

POST /api/social/friends/:requestId/decline

Expected:

status=DECLINED

Không tạo FRIEND relationship.

==================================================
PROOF 4 — FOLLOW USER
=====================

User A follow User B.

POST /api/social/follow/:userId

Expected:

200 OK

Sau đó:

GET /api/social/following

User A thấy User B.

GET /api/social/followers

User B thấy User A.

==================================================
PROOF 5 — UNFOLLOW USER
=======================

DELETE /api/social/follow/:userId

Expected:

200 OK

Sau đó:

followers giảm.

following giảm.

==================================================
PROOF 6 — USER SEARCH
=====================

GET /api/social/search?q=social-user

Expected:

trả về cả User A và User B.

Search phải đọc dữ liệu thật.
